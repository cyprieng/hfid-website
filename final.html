Final summary of Self-Checkout redesign

Or, How I Learned To Stop Worrying And Love The Grocery Store

Background: What Is Self-Checkout?

A recent phenomenon in grocery stores is the introduction of self-checkout machines. Shoppers are able to scan their own groceries, which allows the stores to cut down on labor costs. Self-checkout machines are typically seen at larger chain stores, such as Stop & Shop and Shaw’s. Most systems will have one attendant (the “floater”) monitoring every three or four self-checkout counters. The floater serves as a check against theft, and can help any shoppers who need assistance.

Pain Points

Through observation at various grocery stores (CVS, Shaw’s, and Stop & Shop) that implement self-checkout, we identified four major pain points. These pain points are directly responsible for making self-checkout an unappealing and not streamlined process.

  • Scanning produce is confusing and slow
  • Easy for shoppers to get tripped up by the machine
  • Weighted bagging area leads to many user errors
  • Machine constantly calls the attendant

To read more about the user testing which led to these pain points, read these reports: link to needs analysis and formal usability testing

Our Solution

Physical System Changes

There are four main changes to the physical system:

  • RFID chips to scannable items
  • Image recognition (for produce)
  • Removed weighed bagged system
  • Added return bin (which has RFID sensors and weight sensor)

The RFID chips, image recognition and the return bin all contribute to a more secure system. This is important for the self-checkout floaters and the companies which buy this system. With RFID chips, the floater can guarantee that all scannable items leaving the store have been paid for (or an alarm would go off). Image recognition also ensures shoppers are selecting the correct produce rather than the cheapest one. Finally, the return bin guarantees that canceled items are not stolen.






All of the above physical alterations also make the system more robust/faster for the shopper. The RFID chips guarantee an item is never “double scanned”. Image recognition prevents the user from having to scroll through multiple pages for produce. The weighted bagging system caused most of the distress for shoppers. Many times a shopper didn’t correctly use the bagging system, resulting in the shopper having to wait for an floater to come help. Finally, the return bin adds a place for shoppers to return items - a feature that currently doesn’t exist. It saves time because shoppers don’t have to find a floater or walk back to the item’s original place in the store.

Software System Changes

There are four main changes to the software system:

  • “Call for Help” button on every page
  • Back button on every page
  • Image recognition (but still have keyboard)
  • Cancel item button
  • State of self-checkout at top of page

In order to help the user have more control of the system, a help button was added with the ability to cancel the call. This lets the shopper control the request from beginning to end. Also, the back button lets them “cancel” a button click. The system never holds the shopper to a button click.

In order to enable faster checkout, the image recognition and cancel item button are paramount. Image recognition prevents the user from having to scroll through pages of produce options. However, again to let users keep control there is still the option to type in their produce. (If image recognition doesn’t work.) Finally, letting users cancel their own items ensures they don’t need to wait for a floater.

To ensure clarity of the system, the state of the machine (ex. identify produce, scan item) is at the top of the screen. This subtly helps the shopper if they ever forget what step they are currently performing.
